ロジックマップ・ワークシート

論理的なアカデミック・アーギュメントを組み立てる事前整理ツール
AirPangaea
Global Academic Pass Course
TOMODACHI English
よい議論を書く前に、まずは議論の形を「見える化」することが大切です。ロジックマップは、自分の立場・理由・具体例・反論への応答を、1枚の紙の上に図として整理するツールです。授業で学んだ O-R-E-S スピーキング・テンプレートをさらに一歩進め、「反対意見にも応える」という要素を加えたものと考えてください。文章を書き始める前にマップで思考を整えることで、論点がぶれない、まとまりのあるパラグラフが書けるようになります。
■ なぜロジックマップを使うのか？
① 論理の弱い箇所が早い段階で見える 自分のReasonの下にExampleが書けないなら、その議論はおそらく抽象的すぎます。文章を書き始めてから気づくのではなく、マップの段階で修正できます。
② 反対意見に向き合う構造ができる アカデミックな良い答えは、反対意見を無視しません。あえて「反対側はこう言うだろう」と書き出したうえで、それに対して自分の立場を擁護します。
■ ロジックマップの4つの要素
Thesis主張
自分の中心的な立場を、明確な一文で。
Reason理由
主張を支える、最も強い理由。
Example具体例
一般論ではなく、具体的で現実の状況。
Counter + Response反論 + 応答
反対派の主張と、それでも自分の立場が成り立つ理由。
■ 記入例
サンプル・プロンプト
"Should schools ban smartphones in classrooms?"
記入済みロジックマップ（英語で記入）
THESIS: Schools should ban smartphones during class but allow them during breaks. ├─ Reason: Phones fragment attention even when │ students are not actively using them. │ └─ Example: In my class, a teacher ran a one-week │ phone-free trial. Discussion got noticeably │ more focused — even quieter students spoke up. │ └─ Counter: Phones can be useful learning tools (dictionaries, research, translation). └─ My response: Those uses are real but rare in a typical class. A shared classroom tablet or laptop can handle them without the constant notification pull a phone has.

■ よいロジックマップの条件

  • Thesis（主張）は一文。しかも「誰かが反対できる程度に具体的」であること。誰もが同意する内容は主張として弱い。
  • Example（具体例）は、具体的な場面・授業・経験であること。"in general" や "many people say" のような一般論は避ける。
  • Counter-argument（反論）は、反対派が「その通りだ」と同意してくれるような形で書くこと。"それは違う" と言われるような雑な反論ではダメ。
  • Response（応答）は、反論を聞いたうえで、なぜ自分の主張がそれでも成り立つのかを説明する部分。主張をくり返すだけでは応答になりません。

■ あなたのロジックマップを記入
THESIS: ├─ Reason: │ │ └─ Example: │ │ │ └─ Counter: └─ My response:

■ 提出前のセルフチェック

  • 自分の Thesis に、他の人が反対する可能性はありますか？ 誰もが同意する内容なら、主張として弱いのでもっと鋭くしましょう。
  • Example は、自分の生活・学校・地域で起きた具体的な出来事ですか？「一般的に」「多くの人は」といった表現を避けましょう。
  • Counter-argument は、反対派が実際に言いそうな形で書けていますか？「そうじゃない」と言われる内容だと、反論として成立しません。
  • Response は、反論を踏まえたうえで自分の立場がなぜ成り立つのかを説明していますか？ Thesisの繰り返しになっていませんか？
上手に書くコツ：各欄を埋めるのに長い文章を書く必要はありません。むしろ、短く・具体的な記述のほうが役に立ちます。長文になりそうになったら、一度立ち止まって「自分が本当に言いたいことの核は何か？」と自問し、それだけを書くようにしましょう。
